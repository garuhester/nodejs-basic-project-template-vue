物联网集成方案Rest API规范

一、URI 规范

  1. URI中英文全部小写
  2. 不可包含下划线 “_”防止Nginx自动过滤
  3. 参数列表，中文字符使用URI encode编码

二、Headers

公共请求参数是指每个接口都可能需要传递的参数，公共参数通过header传递。

三、Request

  1. 安全性与幂等性

    • 安全性：不会改变资源状态，可以理解为只读的
    • 幂等性：执行1次和执行N次，对资源状态改变的效果是等价的。
    • 以上五种请求方式满足下列安全性与幂等性要求矩阵。

  2. 请求规范

    1. 查询使用GET

    2. 新增操作使用POST，如新增用户等。

    3. 全量更新且仅修改单个资源，如覆盖用户信息等使用PUT，部分更新使用PATCH

    4. 复杂业务操作，涉及到多资源的情况，使用POST

    5. 删除单个资源，使用DELETE

    6. 请求URL规范

      GET 单个参数使用 PATH 例如 xx/{id} ，多个参数使用 QUERY方式 例如 xx?id=-xx&name=xx

      PUT,POST 使用 application/json 即 BODY 方式传递参数

      DELETE 使用 PATH 方式传递参数 例如 xx/{id}

  3. Body format

    1. 根据业务复杂情况，可自由使用json或者表单方式情况，但必须严格设置正确的Content-Type。比如json：application/json、表单：application/x-www-form-urlencoded。严禁使用application/text
    2. 上传文件使用表单文件上传

四、Response

  1. Response统一使用json
